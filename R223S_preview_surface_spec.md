# R223S preview surface spec

## 1. Preview layout principle

The preview surface must support review, not teaching use.

Recommended information order:

1. Safety banner
2. Sample selector
3. v0.1 / v0.2 candidate comparison summary
4. Teacher default draft preview
5. Review ledger summary
6. Component/screen/sheet/evidence metadata
7. Hold conditions checklist

## 2. Teacher default draft preview

The teacher draft area must prioritize natural manuscript reading:

- no backend field wall
- no component shelf
- no event_id list in default view
- no clickable apply control

Allowed helper labels:

- v0.2 candidate preview
- sample fixture
- no writeback

## 3. Review ledger summary preview

The ledger summary may show:

- unit_phase_role
- practice_intensity
- primary classroom event patterns
- component trigger statuses
- screen/sheet/evidence mapping

The ledger summary should be folded or visually secondary.

## 4. Comparison preview

The comparison preview must answer:

- What changes between v0.1 and v0.2 candidate?
- Did teacher readability improve, hold, or decline?
- Did classroom event depth improve?
- Did review ledger completeness improve?
- Did field leakage risk increase?

## 5. Component trigger preview

Component metadata must not look executable.

Allowed display:

```text
component status: review metadata only
```

Forbidden display:

```text
Run component
Insert component
Open classroom component
Apply to big screen
```
