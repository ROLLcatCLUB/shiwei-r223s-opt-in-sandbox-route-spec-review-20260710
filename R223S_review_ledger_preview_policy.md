# R223S review ledger preview policy

## 1. Policy

Review ledger is required for audit but must not become the teacher default view.

## 2. Default behavior

In sandbox preview:

- teacher draft open by default
- ledger summary folded or secondary
- full ledger available only as review artifact
- no ledger field inserted into teacher draft

## 3. Ledger fields allowed in preview summary

- unit_phase_role
- practice_intensity
- teacher_support_density
- event count
- primary pattern names
- component trigger status count
- screen/sheet/evidence mapping count
- teacher confirmation flags

## 4. Ledger fields not allowed in teacher default view

- activated_adapter_fields
- component_trigger.status
- new_surface_candidate
- unregistered_do_not_execute
- screen_trigger
- learning_sheet_fields
- evidence_outputs

## 5. Hold condition

If the preview surface makes the ledger feel like the main product, R223T must hold for surface reduction.
